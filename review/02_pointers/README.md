## 1. `&` と `*` の基本

```c
int x = 42;
int *p = &x;  // &x : x のアドレスを取得してポインタ p に格納
*p = 100;     // *p : p が指すアドレスの値を読み書きする（デリファレンス）
```

| 演算子 | 意味 | 使用例 |
|--------|------|--------|
| `&`    | アドレス取得（参照） | `&x` → x のメモリアドレス |
| `*`    | デリファレンス（間接参照） | `*p` → p が指す先の値 |

### 重要なポイント

- `int *p` の `*` は「p はポインタ型である」という**型宣言の `*`**
- `*p = 100` の `*` は「p が指すアドレスにアクセス」する**演算子の `*`**  
  → 同じ記号だが役割が異なるので注意
- `p` と `&x` は同じアドレス値を持つ。`*p` と `x` は同じメモリを参照する

```
メモリイメージ:

  アドレス     値
  0x1000  [ 42 ]  ← x
               ↑
  0x2000  [ 0x1000 ]  ← p (x のアドレスを格納)
```

## 2. NULLポインタ

```c
int *null_p = NULL;           // どこも指していないポインタ
printf("%d\n", null_p == NULL); // → 1 (true)
// *null_p = 1;               // ← NG: セグメンテーション違反（クラッシュ）
```

### 重要なポイント

- `NULL` は `(void *)0` として定義されており、「どこも指していない」状態を表す
- NULLポインタをデリファレンスすると**未定義動作 (Undefined Behavior)** となり、通常はクラッシュする
- ポインタを使う前には必ず `NULL` チェックを行うのが安全なコーディングの基本

```c
// 安全なパターン
if (null_p != NULL) {
    *null_p = 1;  // NULL でないことを確認してからデリファレンス
}
```

> **Rust との比較:** Rust の `Option<T>` の `None` に相当するが、Rust はコンパイル時に NULL 安全性を保証するのに対し、C はランタイムクラッシュになる。

## 3. 構造体ポインタと `->` 演算子

```c
typedef struct { int x; int y; } Point;

Point pt = {3, 4};
Point *pp = &pt;

pt.x        // 直接アクセス（ドット演算子）
pp->x       // ポインタ経由アクセス（アロー演算子）
(*pp).x     // pp->x と完全に等価な書き方

pp->x = 10; // ポインタ経由でメンバを変更 → pt.x も変わる
```

### `->` と `(*).` の関係

```
pp->x  ≡  (*pp).x
```

`->` は `(*ポインタ).メンバ` の糖衣構文（シンタックスシュガー）。  
ポインタ経由で構造体メンバにアクセスする際は `->` を使うのが慣習。

### メモリイメージ

```
  pt (スタック上)         pp
  +-------+-------+      +----------+
  | x=3   | y=4   | ←── | &pt      |
  +-------+-------+      +----------+
```

## 4. 値渡しとポインタ渡しの違い

```c
// 値渡し：呼び出し元に影響しない
void try_swap_by_value(int a, int b) {
    int tmp = a; a = b; b = tmp;  // コピーを操作するだけ
}

// ポインタ渡し：呼び出し元の変数を直接変更できる
void swap(int *a, int *b) {
    int tmp = *a; *a = *b; *b = tmp;  // アドレス経由で実体を操作
}
```

### 動作の違い

```
値渡し:
  main の a=1, b=2
        ↓ コピーして渡す
  関数内 a=1, b=2 → swap → a=2, b=1 （コピーが変わるだけ）
        ↓ 関数終了で破棄
  main の a=1, b=2 ← 変わっていない！

ポインタ渡し:
  main の a=1, b=2
        ↓ アドレスを渡す (&a, &b)
  関数内で *a, *b を通じて実体を書き換える
        ↓
  main の a=2, b=1 ← 変わった！
```

### 使い分けの指針

| 用途 | 推奨 |
|------|------|
| 読み取り専用の小さな値 | 値渡し |
| 関数から複数の値を返したい | ポインタ渡し（出力引数） |
| 大きな構造体を渡す（コピーコスト削減） | `const` ポインタ渡し |
| 呼び出し元の変数を変更したい | ポインタ渡し |

## 5. ポインタ演算

```c
int arr[3] = {10, 20, 30};
int *ptr = arr;  // arr は arr[0] のアドレスと等価

*ptr   // → 10 (arr[0])
ptr++  // ptr を 1要素分（= sizeof(int) バイト）進める
*ptr   // → 20 (arr[1])
ptr++
*ptr   // → 30 (arr[2])
```

### 重要なポイント

- `ptr++` は**バイト数ではなく要素数**で進む。`int *` なら `sizeof(int)` バイト（通常4バイト）進む
- 配列名 `arr` は `&arr[0]` と等価。ポインタとして使える
- `arr[i]` は `*(arr + i)` と完全に等価（添字演算子はポインタ演算の糖衣構文）

```
メモリ上のイメージ（int = 4バイトの場合）:

アドレス:  0x100   0x104   0x108
           [ 10 ] [ 20 ] [ 30 ]
             ↑
           ptr（最初）
                    ↑
                  ptr+1（ptr++ 後）
```

### 演算の種類

| 演算 | 意味 |
|------|------|
| `ptr + n` | n 要素先のアドレス |
| `ptr - n` | n 要素前のアドレス |
| `ptr++` / `ptr--` | 1要素分移動 |
| `ptr1 - ptr2` | 2つのポインタ間の要素数 |

> **注意:** 配列の範囲外にアクセスすると未定義動作になる。

## 6. ポインタのポインタ（二重ポインタ）

```c
int val = 99;
int *ptr  = &val;   // ptr は val のアドレスを持つ
int **pptr = &ptr;  // pptr は ptr のアドレスを持つ

**pptr = 77;  // pptr → ptr → val と辿って val を変更
// val == 77
```

### メモリイメージ

```
  val              ptr               pptr
  +------+         +----------+      +----------+
  | 99   | ←───── | &val     | ←── | &ptr     |
  +------+         +----------+      +----------+
  0x1000           0x2000            0x3000

**pptr で val に到達するまでの経路:
  pptr → 0x2000（ptr のアドレス）
  *pptr → ptr の値 = 0x1000（val のアドレス）
  **pptr → val の値 = 99 → 77 に書き換え
```

### 主な用途

- **関数内でポインタ自体を変更したい場合**（例: リスト先頭ノードの挿入）
- **文字列の配列** (`char **argv`) のような多次元データ構造
- **動的メモリ確保の出力引数** (`malloc` 結果を関数外に返す場合)

```c
// 例: 関数内でポインタ変数を変更する
void reset_ptr(int **pp) {
    *pp = NULL;  // 呼び出し元のポインタを NULL にする
}
```

## 7. `const` とポインタの4パターン

`const` の位置によって「何を変更不可にするか」が変わる。

```c
int n = 5;
int m = 10;
```

### パターン一覧

| 宣言 | ポインタ自体の変更 | 指す先の値の変更 | 用途 |
|------|--------------------|------------------|------|
| `int *p` | 可 | 可 | 通常のポインタ |
| `const int *cp` | 可 | **不可** | 読み取り専用参照（入力引数） |
| `int *const pc` | **不可** | 可 | 固定アドレスへの書き込み |
| `const int *const cpc` | **不可** | **不可** | 完全に読み取り専用 |

```c
const int *cp = &n;  // ポインタは変えられる、値は変えられない
cp = &m;             // OK: cp 自体を別アドレスに変更
// *cp = 99;         // NG: コンパイルエラー

int *const pc = &n;  // ポインタは変えられない、値は変えられる
*pc = 99;            // OK: n の値を変更
// pc = &m;          // NG: コンパイルエラー
```

### 読み方のコツ

**右から左に読む**と分かりやすい：

```
const int *cp
→ cp は (int const) へのポインタ → 指す先が const

int *const pc
→ pc は const な (int へのポインタ) → ポインタ自体が const
```

### API 設計での慣用パターン

```c
// 入力：読み取り専用 → const int * を使う
void print_value(const int *p) {
    printf("%d\n", *p);
    // *p = 99; ← コンパイルエラーで誤変更を防げる
}

// 出力：書き込み用 → int * を使う
void set_value(int *p, int val) {
    *p = val;
}
```

> `const int *` を入力引数に使うことで、「この関数は値を変更しない」という意図をコンパイラとコードの読み手に伝えられる。

## まとめ：ポインタ操作チートシート

```c
int x = 42;

// 宣言・初期化
int *p = &x;      // x のアドレスを p に格納

// デリファレンス
int val = *p;     // p が指す先の値を読む
*p = 100;         // p が指す先の値を書く

// NULL チェック
if (p != NULL) { ... }

// 構造体
Point *pp = &pt;
pp->member;       // (*pp).member と同じ

// ポインタ演算（配列）
int *ptr = arr;
ptr++;            // 次の要素へ（sizeof(int) バイト移動）

// 二重ポインタ
int **pp = &p;
**pp = 77;        // 2段階デリファレンスで値を変更

// const パターン
const int *cp;    // 値を変えられない（ポインタは変えられる）
int *const pc;    // ポインタを変えられない（値は変えられる）
```
