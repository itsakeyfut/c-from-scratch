# From Scratch In C

## Build

```sh
clang -o main.exe main.c && ./main.exe
```

### オプション

| フラグ | 内容 |
|--------|------|
| `-Wall -Wextra` | 警告を有効化 |
| `-std=c11` | C11 標準を指定 |
| `-g` | デバッグ情報を付与（gdb / lldb 用） |
| `-fsanitize=address,undefined` | AddressSanitizer + UBSan を有効化 |

開発中の推奨コマンド:

```sh
clang -std=c11 -Wall -Wextra -g -fsanitize=address,undefined -o main.exe main.c && ./main.exe
```

## 進め方
### 1. docs/配下の好みのトピックを選ぶ
ファイル名は「トピック名.md」と定義しています。
ドキュメントの内容は実装するトピックの仕様を記載しています。

### 2. ドキュメントを精読
まずは仕様を理解します。そして、どうやってコードに落とし込むかを考えます。
UB を引き起こす可能性のある注意が必要な操作がある場合はドキュメントの最終行辺りに回避方法を記載しています。

### 3. まずは自分の手で実装してみる
実装の際は AI は絶対に使用しないこと。
ドキュメントに記載している仕様はかなり具体的に書いているため、ちゃんと読めばコードに落とし込むことはできる。

### 4. レビューしてもらう
AI を使って実装したコード（マクロ、関数）をレビューしてもらいます。
例えば次のような指示でレビューしてもらいます：
「str_append を実装しました。レビューしてください。」 

AI がドキュメントを参照して、実装が要件を満たしているか、安全かどうかをレビューします。

### 5. 改善
AI のレビューを参考に修正を行います。
処理でわからないものは深堀りして質問します。
質問した内容は README.md にメモしておきます。

これにより、わからないところだけノートにまとめることができます。
